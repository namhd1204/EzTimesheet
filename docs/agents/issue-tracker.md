# Issue tracker: Local Markdown

Issues and PRDs for this repo live as local markdown files. This is ideal for solo projects or offline work.

## Conventions

- **Storage**: `.scratch/<feature-name>/`
- **Issue file**: `.scratch/<feature-name>/ticket.md`
- **PRD file**: `.scratch/<feature-name>/prd.md`

## When a skill says "publish to the issue tracker"

1. Create the directory `.scratch/<feature-name>/` if it doesn't exist.
2. Write the issue content to `ticket.md`.

## When a skill says "fetch the relevant ticket"

Read the contents of `.scratch/<feature-name>/ticket.md`.
